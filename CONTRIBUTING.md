# Guía de Contribución - Goliat-Orbital

Gracias por tu interés en contribuir a Goliat-Orbital. Este proyecto es el resultado de años de diseño conceptual y documentación técnica. Tu ayuda puede marcar la diferencia para llevarlo a la realidad.

---

## Formas de Contribuir

### 1. Reportar problemas (Issues)

Si encuentras un error en la documentación, una inconsistencia técnica o una oportunidad de mejora:

- Busca primero en los issues existentes para evitar duplicados.
- Usa la plantilla correspondiente (bug report, feature request).
- Incluye información detallada (sección, archivo, descripción del error).

### 2. Sugerir mejoras (Feature requests)

Si tienes una idea para mejorar Goliat-Orbital:

- Describe claramente la funcionalidad propuesta.
- Explica por qué es necesaria (¿mejora la captura? ¿el procesamiento? ¿la logística?).
- Si es posible, menciona cómo podría implementarse y qué tecnologías existen.

### 3. Mejorar la documentación

La documentación es la base del proyecto. Puedes contribuir con:

- Correcciones ortográficas o de formato.
- Clarificaciones de conceptos técnicos.
- Traducciones (inglés, portugués, etc.).
- Ejemplos adicionales o casos de uso.

### 4. Compartir experiencia

Si eres ingeniero aeroespacial, científico de materiales, o trabajas en la industria espacial:

- Tu visión sobre la viabilidad técnica es valiosa.
- Puedes abrir un issue con tu perspectiva.
- Si tienes acceso a datos, papers o patentes relevantes, compártelos.

---

## Estándares de Documentación

### Markdown

- Usar títulos jerárquicos (`#`, `##`, `###`).
- Separar secciones con `---`.
- Para código, usar bloques con ```lenguaje.
- Mantener líneas de máximo 100 caracteres.

### Archivos técnicos

- Cada componente debe tener su propio archivo `.md`.
- Incluir especificaciones técnicas, principios de funcionamiento, y referencias.
- Usar tablas para datos cuantitativos.

### Commits

- Usar mensajes claros en inglés o español.
- Formato: `Tipo: Descripción breve`.
- Tipos comunes: `Fix`, `Feat`, `Docs`, `Style`, `Refactor`.

**Ejemplos:**
```
Fix: Corregir error en la tabla de especificaciones del horno
Docs: Actualizar hoja de ruta con fechas estimadas
Feat: Agregar diseño de la cabina autónoma
```

## Flujo de Trabajo

### Para contribuciones pequeñas (documentación, correcciones)

1. **Fork** del repositorio.
2. **Clonar** localmente.
3. Crear una **rama** con nombre descriptivo (`fix/descripcion-breve`).
4. Hacer cambios y **commit**.
5. **Push** y crear **Pull Request** a la rama `main`.

### Para contribuciones grandes (nuevos módulos, cambios estructurales)

1. Abrir un **issue** primero para discutir la idea.
2. Esperar feedback de los mantenedores.
3. Coordinar para evitar trabajo duplicado.
4. Seguir el mismo proceso de PR.

---

## Pruebas y Validación

Goliat-Orbital es un proyecto conceptual, pero todas las contribuciones técnicas deben estar respaldadas por:

- **Referencias** a papers, patentes o tecnologías existentes.
- **Cálculos** o simulaciones (si aplica).
- **Coherencia** con la arquitectura general del sistema.

---

## Conducta y Respeto

- Ser respetuoso con otros colaboradores.
- Aceptar críticas constructivas.
- Enfocarse en lo técnico, no en lo personal.
- Ayudar a mantener un ambiente inclusivo.
